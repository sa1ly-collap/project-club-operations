## Информационная система учёта и анализа деятельности компьютерного клуба

### 1. Контекст
Учебный проект по проектированию информационной системы для операционного учёта и анализа работы компьютерного клуба.  
Фокус — процессы учёта игровых сессий, выручки и загрузки оборудования, которые сейчас ведутся частично вручную и в разных источниках.

### 2. Цель и задачи
**Цель.** Спроектировать информационную систему, которая консолидирует данные по деятельности клуба и снижает долю ручных операций.

**Задачи:**
- Описать текущие процессы учёта и отчётности клуба с помощью BPMN (as‑is).
- Выявить проблемы: дублирование ввода данных, разрозненность отчётов, зависимость от ручного учёта.
- Разработать целевую модель процессов (to‑be) с учётом внедрения информационной системы.
- Определить основные функции ИС и требования к данным (что хранить, какие отчёты нужны).
- Сформировать план внедрения системы (этапы, сроки) с помощью диаграммы Ганта.

### 3. Модели и диаграммы
В проекте использованы следующие артефакты:
- BPMN‑диаграмма текущих процессов учёта и отчётности «as‑is».
- BPMN‑диаграмма целевых процессов «to‑be» с использованием информационной системы.
- UML use case‑диаграмма для описания сценариев работы пользователей с системой.
- Диаграмма Ганта в MS Project с этапами анализа, проектирования и внедрения.

*(сюда можно так же, как в первом проекте, добавить ссылки на картинки: `![BPMN as‑is](имя_файла.png)` и т.п.)*

### 4. Результаты
Проанализирована текущая схема учёта, где значительная часть операций выполняется вручную и данные хранятся в разрозненных файлах.  
Целевая модель процессов и проект информационной системы показывают, как можно сократить долю ручных операций, уменьшить риск ошибок в отчётности и упростить доступ к данным по выручке, загрузке оборудования и активности клиентов.

### 5. Используемые инструменты
- BPMN — описание процессов учёта и отчётности as‑is и to‑be.
- (При наличии) UML — описание пользовательских сценариев и функциональных требований к системе.
- MS Project — диаграмма Ганта с этапами проекта по внедрению ИС.
- Miro / другие средства визуализации — работа с диаграммами и согласование схем.
